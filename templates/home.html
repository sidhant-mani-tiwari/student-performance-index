MODULE 01 · Virtual Environments
Your private kitchen
Your system Python is a shared kitchen. A virtual environment is your own isolated workspace. Every project gets one — no version conflicts, no cross-contamination between projects.
MODULE 02 · Package Management
The recipe, not the kitchen
requirements.txt lists packages from PyPI. setup.py makes your own code importable. pip install -e . wires them — pip reads setup.py internally.
environment setup
# Step 1 — create and activate
python -m venv venv
source venv/bin/activate          # Mac/Linux
venv\Scripts\activate             # Windows

# Step 2 — install external packages
pip install -r requirements.txt

# Step 3 — make your OWN code importable
pip install -e .                  # pip reads setup.py — never run it directly
setup.py — the bridge between your code and Python's import system
from setuptools import find_packages, setup

setup(
    name="my_ml_project",
    version="0.1.0",
    packages=find_packages(),   # finds every folder with __init__.py
)

# After pip install -e . this works from anywhere:
from src.components.data_ingestion import DataIngestion
Every project gets its own virtual environment. venv/ and egg-info/ go in .gitignore. You commit the recipe, not the kitchen.

module 04 — cross-platform path handling
from pathlib import Path
import os, pandas as pd

# ❌ Breaks on Windows — hardcoded slash in a string
path = "artifacts/" + "train.csv"

# ✅ pathlib — / is a Python operator, adapts to any OS
path = Path("artifacts") / "train.csv"

# ✅ __file__ always resolves relative to THIS script's location
#    regardless of where you run it from in the terminal
base_dir  = Path(__file__).parent.parent.parent
data_path = base_dir / "notebooks" / "data" / "stud.csv"
df        = pd.read_csv(data_path)

# Always create directories before saving files
artifact_dir = os.path.dirname(self.ingestion_config.train_data_path)

Engineering Patterns
The senior way of organising code.

Three patterns that prevent spaghetti at scale.

Module 06 · OOP
Object-Oriented Programming
Classes bundle related data and logic into one self-contained unit. __init__ runs automatically on creation. self is the object referring to its own data — like saying "my".
module 06 — a component as a class
class DataIngestion:
    def __init__(self):
        # runs automatically when you do: ingestion = DataIngestion()
        # self = "my" — this object's own train_path
        self.train_path = Path("artifacts") / "train.csv"
        self.test_path  = Path("artifacts") / "test.csv"

    def run(self):
        df = pd.read_csv(data_path)
        df.to_csv(self.train_path)          # "my train_path"
        return self.train_path, self.test_path

ingestion             = DataIngestion()     # instance — real object from blueprint
train_path, test_path = ingestion.run()
Module 07 · Dataclass
The Config Pattern
Config answers WHERE. Logic answers HOW. @dataclass auto-generates __init__ so you just declare fields. Changing a file path never requires touching your logic.
module 07 — separating where from how

# Answers only WHERE — zero logic inside

    train_data_path: str = os.path.join('artifacts', 'train.csv')
    test_data_path:  str = os.path.join('artifacts', 'test.csv')

# Answers only HOW — zero hardcoded paths
class DataIngestion:

Module 08 · Utils
The Utility Pattern — DRY
If you write the same logic twice, it belongs in utils.py. Don't Repeat Yourself. save_object() and load_object() are written once — used by every component that needs them.
module 08 — utils.py with dill serialization
import dill   # extends pickle — handles lambdas and custom classes

def save_object(file_path: str, obj) -> None:
    # Used by: DataTransformation (preprocessor), ModelTrainer (model)
    os.makedirs(os.path.dirname(file_path), exist_ok=True)
    with open(file_path, "wb") as f:   # "wb" = write binary
        dill.dump(obj, f)

def load_object(file_path: str):
    # Used by: PredictionPipeline to load model + preprocessor
    with open(file_path, "rb") as f:   # "rb" = read binary
        return dill.load(f)
Config answers WHERE. Logic answers HOW. Utils removes duplication. Three patterns — every senior ML codebase you'll ever read uses all three.

Observability & Reliability

The safety net around every pipeline run.

MODULE 09 · Logging
The Black Box Diary
Persistent, timestamped record of every event. New file per run. INFO for normal flow. ERROR for crashes. print() disappears — logs survive on disk.
MODULE 10 · Custom Exceptions
The Crash Detective
Not just what crashed — exactly which file and line. sys.exc_info() extracts the traceback. Your error message is instantly actionable instead of a wall of text.
module 09 — logger.py
import logging, os
from datetime import datetime

# New timestamped log file for every pipeline run
LOG_FILE      = f"{datetime.now().strftime('%m_%d_%Y_%H_%M_%S')}.log"
logs_dir      = os.path.join(os.getcwd(), "logs")
os.makedirs(logs_dir, exist_ok=True)
LOG_FILE_PATH = os.path.join(logs_dir, LOG_FILE)

logging.basicConfig(
    filename=LOG_FILE_PATH,
    format="[%(asctime)s] %(lineno)d %(name)s - %(levelname)s - %(message)s",
    level=logging.INFO,   # INFO and above — filters out DEBUG noise
)
module 10 — exception.py — the crash detective
def get_error_details(error, error_detail: sys) -> str:
    _, _, exc_tb = error_detail.exc_info()
    filename     = exc_tb.tb_frame.f_code.co_filename  # file
    line_number  = exc_tb.tb_lineno                    # line
    return f"Error in [{filename}] at line [{line_number}]: [{error}]"

class CustomException(Exception):
    def __init__(self, error_message, error_detail: sys):
        super().__init__(str(error_message))
        self.error_message = get_error_details(error_message, error_detail)
        logging.error(self.error_message)    # writes to log immediately

    def __str__(self):
        return self.error_message            # controls what gets printed

# Usage — same pattern in every component:
except Exception as e:
    raise CustomException(e, sys)
The last INFO log before an ERROR tells you exactly how far the pipeline got before it crashed. That's your black box doing its job.

module 11 — data ingestion saves three artifacts
@dataclass
class DataIngestionConfig:
    raw_data_path:   str = os.path.join('artifacts', 'raw.csv')
    train_data_path: str = os.path.join('artifacts', 'train.csv')
    test_data_path:  str = os.path.join('artifacts', 'test.csv')

def initiate_data_ingestion(self):
    df = pd.read_csv(data_path)
    df.to_csv(self.ingestion_config.raw_data_path, index=False)  # checkpoint
    train_set, test_set = train_test_split(df, test_size=0.2, random_state=42)
    # random_state=42 — same split every single run, reproducible
    train_set.to_csv(self.ingestion_config.train_data_path, index=False)
    test_set.to_csv(self.ingestion_config.test_data_path,   index=False)
    return self.ingestion_config.train_data_path, self.ingestion_config.test_data_path

training pipeline — the manager calling workers in sequence
class TrainingPipeline:
    def run_pipeline(self):          # no __init__ needed — no config, no artifacts
        try:
            ingestion              = DataIngestion()
            train_path, test_path  = ingestion.initiate_data_ingestion()

            transformation         = DataTransformation()
            train_arr, test_arr, _ = transformation.initiate_data_transformation(
                                         train_path, test_path)

            trainer                = ModelTrainer()
            r2_score               = trainer.initiate_model_trainer(train_arr, test_arr)
            return r2_score
        except Exception as e:
            raise CustomException(e, sys)
prediction pipeline — inference at request time
class PredictPipeline:
    def predict(self, features):
        model        = load_object(self.predict_config.model_path)
        preprocessor = load_object(self.predict_config.preprocessor_path)

        # transform() only — preprocessor is already fitted from training
        # calling fit_transform() here would be data leakage
        transformed  = preprocessor.transform(features)
        return model.predict(transformed)
